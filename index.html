Development Methodology

The project followed an iterative approach in 4 main phases:

  1. Physical Modeling (4 weeks): Development of thermodynamic model in Simscape with validation at 3 main operating points
  2. ML Training (3 weeks): Synthetic dataset generation (10K samples), feature engineering, neural network training and validation (3 layers, 128-64-32 neurons)
  3. Experimental Validation (5 weeks): Test bench testing in 25 different operating conditions, including 8 simulated fault configurations
  4. GUI Development (2 weeks): User interface development with real-time visualization, alert system, and PDF report export

Dataset and Training

  • 10,000 synthetic scenarios generated (70% nominal, 30% anomaly conditions)
  • Features: 24 variables (temperatures, pressures, powers, efficiency)
  • Training: 70% train, 15% validation, 15% test
  • Epochs: 200 with early stopping (patience=20)
  • Optimizer: Adam (lr=0.001)
  • Loss: MSE for regression, Cross-Entropy for classification

Technical Challenges Addressed

‚öñÔ∏è Imbalanced Data Handling
Problem: Anomalies represent < 2% of real data, causing bias toward negative predictions.
Solution: Implemented hybrid approach with synthetic minority oversampling (SMOTE), dynamic class weights, and post-training threshold tuning. Result: F1-score improved from 0.65 to 0.90.
üå°Ô∏è Operating Conditions Variability
Problem: Very different operating patterns between summer/winter, day/night, causing false positives.
Solution: Developed context-aware thresholding system with 8 different operating profiles. Additional features: season, time of day, outdoor temperature. 60% reduction in false positives.
üîå Scalability and Performance
Problem: System must handle 25+ units with 50+ sensors each at 1Hz sampling rate.
Solution: Asynchronous architecture with async/await, connection pooling, data batching. Distributed processing with Ray for parallel training. Monitoring: < 5% CPU usage per monitored unit.

Types of Anomalies Detected

  • Performance Degradation: Gradual reduction in efficiency (COP) over time
  • Sensor Faults: Erroneous or out-of-range readings from malfunctioning sensors
  • Refrigerant Leakage: Gradual leaks detected through pressure trend analysis
  • Fouling: Dirt accumulation on heat exchangers reducing efficiency
  • Compressor Issues: Anomalies in compressor behavior
  • Control Malfunctions: Anomalous control system behavior
